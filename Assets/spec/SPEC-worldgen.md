# World Generation

All world gen lives in `WorldGen.cs` (pure static class, no MonoBehaviour). Called from `WorldController.GenerateDefault()` with a random seed.

## Pipeline

1. **Surface heightmap** — FBM Perlin noise sampled at a **domain-warped x** (the column index is shifted by a second noise at similar freq before the main FBM lookup) with **region-level amplitude variation** (a third, much slower noise multiplies the surface amplitude per column, so some stretches read as plains and others as hills/mountains). Result is clamped to `[SurfaceMin, SurfaceMax]`. Domain warping breaks the uniform "wavy" feel of vanilla FBM; amplitude variation adds large-scale character (calm vs dramatic regions). Spawn zone forced flat with smoothstep blend at edges. Single-column nubs smoothed out.
2. **Fill terrain** — Dirt layer on top (`DirtDepth` tiles), limestone below (default stone variant).
3. **Stone veins** — `ApplyVeins` runs two passes (granite, slate). Each samples its own FBM field and converts limestone tiles to the vein tile where `noise < threshold × depthBias`. `depthBias` is 1.0 at the vein's `DepthCenter` (0=surface, 1=bedrock), tapering linearly to 0 at ±`DepthWidth`. Only limestone tiles are touched, so veins don't overlap (first pass wins). Granite is mid-depth biased; slate is deep. Seeds are offset per-pass for reproducibility without correlation between vein types.
4. **Cave noise field** — 2D FBM (2 octaves, persistence 0.5, lacunarity 2) stored as continuous floats, normalized to ~[0,1]. Spawn zone = 1.0 (solid), exclusion zone near surface = 0.4 (blocks natural caves but worms can push through), normal underground = FBM sample.
5. **Worm tunnel blending** — Each world rolls a worm count in `[WormCountMin, WormCountMax]`. For each worm, a random start column is picked from the eligible set (away from edges and outside the spawn buffer), with `WormMinSeparation` enforced between starts (relaxed if no candidate satisfies). The worm walks from surface downward with a persistent horizontal direction (small chance to reverse per step) and **reflects off the spawn-zone buffer**, so even after random reversals it stays on its starting side and never tunnels into the player's starting area. `SoftCarve` lowers noise values with smoothstep falloff along the path, merging naturally with surrounding Perlin caves.
6. **Threshold** — Continuous field → boolean mask. Depth-varying threshold (stricter near surface, looser deep). Covers full underground including exclusion zone.
7. **CA smoothing** — 1 round of cellular automata (>4 solid neighbors → solid). Kept minimal so FBM detail survives into the final mask; bump up if caves feel too noisy. Then a pass to remove 1-wide horizontal outcroppings.
8. **Small cave removal** — BFS flood-fill removes cave regions smaller than `MinCaveSize`.
9. **Apply caves** — Boolean mask → set tiles to empty. Caves carve through whichever stone variant is there (veins don't get special treatment).
10. **Background walls (`SetBackgrounds`)** — Place a wall behind every tile below `surfaceY[x]` with a near-surface skylight relaxation (non-solid tiles within 2 of the surface stay open so shallow caves visibly punch through to sky). Pass 2 erodes a 1-tile peel at every sky/cave boundary so the wall isn't directly behind the topmost solid row (keeps surface and cave-edge brightness). Wall type is positional: top `DirtDepth` rows = Dirt, deeper = Stone. Saved per-tile, never changes after mining. Runs inside `Generate` (rather than later in `WorldController.GenerateDefault`) so step 11 can read post-erosion `backgroundType`.
11. **Free-floating chunk removal (`RemoveFloatingChunks`)** — BFS flood-fill from every connectable cell on the bedrock row (`y=0`), where a cell is "connectable" iff it has a solid tile OR a non-`None` `backgroundType`. 4-connected. Any solid tile not reached is set to empty; its `backgroundType` is left alone (a stranded wall under newly-bared air just reads as a cave wall). Catches the floating-spire artefact where a worm tunnel + sharp surfaceY transition leaves a small solid cluster whose wall got eroded by Pass 2, visually disconnecting it from the mainland.
12. **Cave water (`FillCaveWater`)** — Each cave region (4-connected flood-fill of the `isCave` mask, same shape as `RemoveSmallCaves`) rolls `CaveWaterChance` and on hit fills a random fraction `[0, 1]` of its volume with water at a binary-searched uniform level (partial-rim cells topped up in ascending-x order so the remainder still reads as intentional). Open caves (worm-tunneled to surface) aren't sealed off — the post-Generate liquid settler drains them before the player sees the world.
13. **Depression filling** — "Trapping rain water" algorithm finds surface basins. Eligible basins (volume ≥ 1, not a worm chimney, floor tile solid) are collected via `BasinShouldBeRejected` (one column with `surfaceY - effSurfaceY > PitRejectDepth` *or* a non-solid tile directly below the basin floor rejects the whole basin), then bucketed by basin center column into `WaterChunkCount` equal-width horizontal chunks. Each chunk independently rolls a budget in `[WaterChunkBudgetMin, WaterChunkBudgetMax]` and allocates it via a **chunk-global water level**: filter to basins with `volume ≥ MinPoolVolume`, then binary-search the highest level `L` such that the total fill volume across all eligible basins — each capped at its own natural rim, via `ChunkVolAtLevel` — stays ≤ budget. Every eligible basin fills to `min(L, basin.naturalRim)`, so water spreads across the chunk like a water table rather than letting one deep basin hog the whole budget. Per-basin rim caps prevent visual overspill. Fallback (no eligible basins, or binary search settled at the lowest floor): partial-fill the deepest basin via `FillBasinUpTo(b, budget)` so the chunk doesn't read as dry. Returns a per-chunk `bool[] chunkFilled` flag the next step consumes.
14. **Dry-chunk pool carving (`CarveDryChunkPools`)** — For each chunk flagged dry by step 13, scan its columns for a `PoolCarveWidth`-wide window with the lowest rim and roughness ≤ `PoolCarveMaxRoughness`, skip windows overlapping the spawn buffer, verify each column's per-column carve floor sits on solid ground, then carve a *stepped trapezoidal* basin (centre flat at `PoolCarveDepth`, edges step down by 1 per column toward the rim, clamped to depth ≥ 1) and flood it. Depth profile formula: `colDepth = clamp(depth - max(0, dist - (half - depth + 1)), 1, depth)`. For `width=5, depth=2` that's `(1, 2, 2, 2, 1)`; for `width=7, depth=3` it's `(1, 2, 3, 3, 3, 2, 1)`. No loose fallback — if no window meets the roughness gate the chunk is logged and skipped, since accepting a high-roughness window means the rim sits above some column's surface and water leaks down a worm tunnel.
15. **Beach sand + clay banks** — `ApplyBeachSand` converts a clumpy ~40% of dirt tiles touching water (orthogonal OR diagonal) into sand. A single-octave Perlin mask (`SandFreq` ≈ 0.18, threshold ≈ 0.52) decides which eligible tiles flip, so beaches form as patches and dunes rather than a uniform sand ring around every pool. Immediately after, `ApplyClayBanks` runs the same pass with its own seed offsets and Perlin mask (`ClayFreq`, `ClayThreshold`), checking `t.type == dirt` so sand-flipped tiles are skipped — sand wins overlap, clay only catches dirt the sand pass didn't take. Both run after `FillDepressions` (water placement must be final) and before `PopulateOverlays` (so grass bits are seeded on dirt's final boundary — though the Tile.type setter would clear them on a later type-flip anyway).
16. **Tile overlays** — `PopulateOverlays` walks every overlay-bearing tile (today: dirt) and seeds `tile.overlayMask` bits on each cardinal edge whose neighbour is non-solid AND non-flooded. This is what makes surface dirt "have grass on top" by default and dirt walls of caves "have grass on the cave-facing side." Runs after depression filling so submerged dirt doesn't get grass on the under-water side. Mining never adds bits, so newly exposed sides stay bare. See SPEC-rendering "Tile overlays" for the rendering side.

**Post-Generate (in `WorldController.GenerateDefault`)** — `WaterController.Settle(maxSteps, moveThreshold)` runs the live CA `SimulateStep` up to `LiquidSettleMaxSteps` times (early-exit when a step transfers ≤ `LiquidSettleMoveThreshold` total water units). Drains cave pools whose worm tunnels exit to the surface, equalizes carved-pool seams, and chips away the diff-1 staircases pass 3 needs a few steps to flatten. Called before market/plant placement so `ScatterPlants`' water-presence check sees the settled topology.

## Key design choices

- **Continuous noise field** for caves (not immediate boolean) so worm tunnels blend with natural caves instead of looking stamped on.
- **FBM + minimal CA** — layering multiple octaves of Perlin adds high-frequency wall detail (pockets, niches, rough edges); CA is deliberately capped at 1 round so that detail survives into the final mask instead of being smoothed back into blobs.
- **Exclusion zone** uses an intermediate noise value (0.4) — high enough to block natural caves, low enough that worm strength can punch through. This lets worms bore from surface into caves while keeping the near-surface layer solid otherwise.
- **Chunked water budget** — the world is split into `WaterChunkCount` equal-width horizontal slices, each rolling its own water budget in `[WaterChunkBudgetMin, WaterChunkBudgetMax]` and allocating it to its own basins (deepest-first). Keeps surface water reasonably even across the map rather than letting one lumpy side soak up everything. Per-chunk randomized budgets give controlled variation in total surface water without the wide swings the older top-N-by-volume scheme produced. Each chunk's "fill at least one basin" fallback ensures no chunk goes dry when basins exist there; `CarveDryChunkPools` carves a stepped trapezoidal basin in any chunk that ended up with no natural water at all, so every chunk reliably ships with a pool.
- **Budget overshoot over partial-fill** — `FillDepressions` always fills each basin to its full natural volume; the rolled chunk budget is a "stop after this one" signal, not a hard cap on the last basin. The old binary-searched partial-fill kept the total close to budget but produced 1-cell puddles whenever the budget landed near zero. Letting the chunk overshoot by one basin's worth is a better trade — total water still varies in a controlled band, and every pool that gets touched looks intentional.
- **Basin reject = chimney + floor sanity** — the surface basin scan rejects a basin if any column either (a) has `surfaceY - effSurfaceY > PitRejectDepth` (a worm-carved chimney that would fill with a tower of water rather than a surface puddle) or (b) has a non-solid tile directly below the basin floor (the 1-tile leak guard — almost always true by `ComputeEffectiveSurface`'s invariant, but catches y=0 bedrock cases and splits over-wide basins at any column where a sub-surface cave nicks the floor). The older "scan 3 tiles below for any cave" check was too strict — caves further below a solid floor don't actually drain a pond sitting on top.
- **Uniform-level cave fill** — `FillCaveWater` binary-searches the highest fill level whose volume is ≤ the rolled target rather than greedily stamping the lowest N tiles. Worlds start paused under the F9 debug panel and snapshot tests, so the initial visual matters: a flat rim reads as a settled pool, while a stacked column would only look right after a few `WaterController` ticks. Open caves draining via worm tunnels is intentional — keeps the pipeline order simple and the resulting overflow into low surface basins is rare.

## Tuning

All tuning params live on a `WorldGenConfig` ScriptableObject at `Assets/Resources/WorldGenConfig.asset`. `WorldGen` exposes a lazy `config` accessor (`Resources.Load`, with a fallback to `ScriptableObject.CreateInstance` + LogError if the asset is missing). Field reads use `WorldGen.config.<FieldName>` from external callers; internal methods bind a local `var cfg = config;` at the top and dereference through it.

Asset is created via **Window → Shonei → Create WorldGen Config asset** (or right-click in Project view → Create → Shonei → WorldGen Config and move into `Assets/Resources/`). Section headers, sliders, and tooltips are declared on the SO fields via `[Header]` / `[Range] / `[Tooltip]` attributes — Unity's default inspector renders them all.

To add a new stone type for vein generation, add the params to `WorldGenConfig.cs` and append another `ApplyVeinPass` call in `ApplyVeins`. Inside the pass: `{Stone}Threshold` gates per-tile conversion (higher = more vein tiles at peak), `{Stone}DepthCenter` and `{Stone}DepthWidth` set the depth band, `{Stone}Freq` controls vein shape (higher = smaller/tighter veins).

## Dev tooling

**WorldGenConfig inspector** (`Assets/Editor/WorldGenTuner.cs` → `WorldGenConfigEditor`) is the single tuning surface. The custom editor renders the default sliders/tooltips plus a "Regenerate (same seed)" / "Regenerate (new seed)" pair + last-seed readout at the top. Inspector edits persist directly to the asset (auto-saved by Unity, including in Play mode), so tuned values survive recompile without any EditorPrefs dance. Multiple presets = multiple duplicate assets.

Regen pipeline: the buttons set `WorldController.pendingSeedOverride` and call `SaveSystem.instance.LoadDefault()` (ClearWorld → ResetSystemState → GenerateDefault → PostLoadInit). `GenerateDefault` consumes the override on its next run, then falls through to a fresh `UnityEngine.Random.Range(...)` for any subsequent in-game new-world calls. `GenerateDefault` also ends with `TimeController.instance?.Pause()` so a freshly generated world starts paused — gives you a still frame to inspect before anything moves.

`SaveSystem.PostLoadInit` also calls `WaterController.UpdateSurfaceMask()` explicitly. Without it, paused starts don't render water (tick-driven updates are gated by the time scale) and same-size regens skip `OnWorldAllocated` so the auto-rebuild path never fires.
